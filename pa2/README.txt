The idea behind our implementation is pretty simple: it is to never operate directly on the Treap itself. As such, whenever the original implementation contained root = doSomething(), we made a local copy of the Treap and did our operations on that. As such, we would not run into out-of-sync issues because only one action would be performed on that copy at a time. In the original implementation, whenever any change was made, all the other operations that were being performed at the same time would be rolled back because the Treap object had been changed. Only when we actually made a change to the object would we write back to the original copy. We noticed that originally even in the case where root was not being changed, we would reassign root = root. As such, we would check that only in the case where root != valueReturned would be perform a write. This would, at the very least, save one write. Our implementation should work because we really don’t make any changes to the logic of the original code. We only extract local copies and perform all the operations on them and only when it requires a change to the original Treap, we write to the root object.
